## Key real bugs caught and fixed during verification (not just assumed working)
- `[EmailAddress]` validation failing on empty (non-null) string — fixed by making Email nullable.
- `FinancialYearId` unconditionally required, breaking every Student edit — fixed to validate only on create.
- `StudentService.DeleteAsync` / `StaffService.DeleteAsync` are soft-deletes (IsActive=false) — UI reworded "Delete" → "Deactivate" to match.
- `InventoryService.DeleteProductAsync` is soft-delete but `GetProductsAsync` hardcodes `IsActive=true` filter — deleted products become permanently invisible from the UI even though the DB row survives. Kept honest "Delete" wording.
- Model-binding mismatch: `SaveCategory(BookCategoryFormModel form)` parameter named `form` didn't match the Razor `CategoryForm` prefix — silently failed to bind. Fixed by renaming parameter to `categoryForm`.
- Placed an HTML `<div>` inside a `<script>` block by mistake (prefill data carrier) — caught before testing, moved outside.
- `Microsoft.OpenApi` 3.8.0 broke the build entirely (source generator incompatibility) — found before it shipped.
